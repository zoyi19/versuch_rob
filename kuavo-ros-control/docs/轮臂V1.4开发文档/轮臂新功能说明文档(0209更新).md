# 轮臂新功能说明文档

该文档在 "轮臂控制API.md" 文档的基础上, 介绍在 2026.02.09 完成的轮臂新功能说明

## 目录

### 相关文件目录

1. src/demo/test_kuavo_wheel_real/timedCmd_example 中的全部 python 脚本, 执行按最优时间执行的轨迹
2. src/demo/test_kuavo_wheel_real/ruckig_setting_test.py 脚本, 用于调整和最优时间相关参数 (同步, 最大速度等)

### 02.09后轮臂代码的新特性说明

1. 删去 /mobile_manipulator_get_mpc_control_mode 切换模式的功能, 一直以 baseArm 模式, 支持所有指令的执行, 安全性降低, 易用性提高;  
2. 旧版本中, 切换模式会自动复位的功能, 调整为单独的服务 /mobile_manipulator_reset_torso, 需要重置时, 请在代码中主动调用; 
3. 新增服务 /mobile_manipulator_get_torso_initial_pose, 可用于获取躯干位姿;
4. 新增服务 /mobile_manipulator_set_ruckig_planner_params, 可用于修改规划器的物理限制 (最大/最小速度, 最大/最小加速度, 最大急动度);
5. 新增服务 /mobile_manipulator_timed_single_cmd, 用于下发但规划器的指令向量和期望执行时间 (受规划器的物理限制影响, 见上一条);  
6. 新增话题 /mobile_manipulator_focus_ee, 用于调整躯干和手臂末端同时执行笛卡尔指令时, 两者的优先级设定;
7. 修改切换 auto_swing 模式的重置手臂, 改为固定时间为1秒的插值运动, 而非最快速度返回;
8. 规划器和控制器分离, MPC 的 observation 中除了浮动基座(x, y, yaw), 其他数据采用 MPC 的输出作为其输入, 误差分析更简单;

## 新增 ROS 的接口说明

### 发布话题 (Published Topics)

#### 1. `/mobile_manipulator_focus_ee` - 笛卡尔跟踪关注优先级调整

**消息类型：** `std_msgs::Bool`

**功能描述：** 

发送是否更关注手臂末端的控制,   
true: 表示更关注手臂末端笛卡尔的跟踪, 允许手臂指令扭曲躯干指令;  
false: 表示更关注躯干笛卡尔的跟踪, 不允许手臂指令扭曲躯干指令;

**注意事项：** 

1. 机器人启动时, 默认为更关注手臂末端跟踪；

**字段说明：**
```yaml
bool data  # True: 表示关注双臂末端跟踪, 
           # False: 表示关注躯干笛卡尔跟踪
```

**相关演示脚本：** 

`timedCmd_example/cmd_arm_ee_world_test.py`  
`timedCmd_example/cmd_arm_ee_local_test.py`

## ROS 服务接口

### 调用服务 (Called Services)

#### 1. `/mobile_manipulator_get_torso_initial_pose` - 获取机器人初始状态的躯干相对baselink的位姿

**服务类型：** `kuavo_msgs::getLbTorsoInitialPose`

**功能描述：** 
启用或禁用手臂快速模式，
1. 启用下肢则下肢电机直接获取lb_leg_traj的指令，不从MPC获取下肢电机指令
2. 启用上肢则手臂电机直接获取kuavo_arm_traj的指令，不从MPC获取手臂电机指令

**请求字段：**
```yaml
bool isNeed       # 设置为 True, 表示需要获得初始化时, 躯干相对baselink的位姿
```

**响应字段：**
```yaml
bool result      # 为 True, 说明服务执行成功
string message   # 详细消息, 在该服务有反馈, 可以打印查看具体消息
geometry_msgs/Vector3 linear:   # 位置向量, xyz, 单位: m
  float64 x
  float64 y
  float64 z
geometry_msgs/Vector3 angular:  # 姿态向量, 采用欧拉角表示, 单位: rad
  float64 x
  float64 y
  float64 z
```

**相关演示脚本：** 
- `timedCmd_example/cmd_torso_pose_test.py`

#### 2. `/mobile_manipulator_reset_torso` - 重置躯干到初始位姿

**服务类型：** `std_srvs/SetBool`

**功能描述：** 请求为 True 时, 机器人开始执行复位, 并反馈复位所需时间, 在复位时间内不允许其他指令打断

**请求字段：**
```yaml
bool data    # 设置为 True, 表示需要机器人躯干重置到初始位姿
```

**响应字段：**
```yaml
bool success        # 是否正在执行重置操作
string message      # 此处是, 反馈浮点时间的字符串, 读取时需要将其重新解析为浮点型
```
**相关演示脚本：** 
- `lb_ctrl_api.py` 中 `reset_torso_to_initial()`

#### 3. `/mobile_manipulator_set_ruckig_planner_params` - 设置不同指令规划器的调度设置

**服务类型：** `kuavo_msgs/setRuckigPlannerParams`

**功能描述：** 用于设置轮臂机器人支持的不同指令的调度设置, 同步/异步, 最大/最小速度, 最大/最小加速度, 最大急动度

**请求字段：**
```yaml
int8 planner_index          # 规划器索引, 具体看下面说明

float64[] velocity_max      # 最大速度的向量, 必须与规划器自由度匹配, 否则设置失败, 会反馈错误信息
float64[] velocity_min      # 最小速度的向量, 可不设置, 不设置则以 -velocity_max 作为设置

float64[] acceleration_max  # 最大加速度的向量, 必须与规划器自由度匹配, 否则设置失败, 会反馈错误信息
float64[] acceleration_min  # 最小加速度的向量, 可不设置, 不设置则以 -acceleration_max 作为设置

float64[] jerk_max          # 最大急动度的向量, 必须与规划器自由度匹配, 否则设置失败, 会反馈错误信息

bool is_sync                # 是否需要同步, 设置同步则所有的指令向量, 会刚好同一时间到达期望位置, 建议底盘设置不同步, 其他指令设置同步
```

**响应字段：**
```yaml
bool result     # 返回为 true, 则表示设置成功
string message  # 该服务中, message 会反馈具体信息, 可打印查看设置失败的原因
```

**规划器配置索引设置**  

- **索引 0**: 底盘位置运动, 自由度为 3, 分别表示 x, y, yaw, 各维度单位分别为: m, m, rad;  
- **索引 1**: 底盘速度运动, 自由度为 3, 分别表示 x, y, yaw, 各维度单位分别为: m, m, rad;  
- **索引 2**: 躯干笛卡尔运动, 自由度为 4, 分别表示 x, z, yaw, pitch, 各维度单位分别为: m, m, rad, rad;  
- **索引 3**: 下肢关节运动, 自由度为 4, 分别表示下肢4个关节的角度, 各维度单位为: rad;
- **索引 4**: 左臂笛卡尔运动, 自由度为 6, 分别表示左臂的6d位姿, 顺序为: x,y,z,yaw,pitch,roll, 各维度单位为: m, m, m, rad, rad, rad;
- **索引 5**: 右臂笛卡尔运动, 自由度为 6, 分别表示右臂的6d位姿, 顺序为: x,y,z,yaw,pitch,roll, 各维度单位为: m, m, m, rad, rad, rad;
- **索引 6**: 左臂上肢关节运动, 自由度为 7, 分别表示左臂7个关节的角度, 各维度单位为: rad;
- **索引 7**: 右臂上肢关节运动, 自由度为 7, 分别表示右臂7个关节的角度, 各维度单位为: rad;

**相关演示脚本：** 
- `lb_ctrl_api.py` 中 `set_ruckig_planner_params()`
- `ruckig_setting_test.py`

#### 4. `/mobile_manipulator_timed_single_cmd` - 设置单个规划器的基于时间的期望指令向量

**服务类型：** `kuavo_msgs/lbTimedPosCmd`

**功能描述：** 用于调用轮臂机器人不同规划器来规划最优时间轨迹, 并执行对应轨迹的跟踪

**请求字段：**
```yaml
int8 planner_index   # 规划器索引, 具体看下面说明, 与 /mobile_manipulator_set_ruckig_planner_params 不相同

float32 desireTime   # 期望执行时间, 单位: s, 如果 desireTime 在物理约束内, 则严格按 desireTime 设置轨迹

float64[] cmdVec    # 指令向量, 必须与所调用的规划器的自由度一致, 否则会报错
```

**响应字段：**
```yaml
bool isSuccess      # 指令下发是否成功, True 表示下发成功

float32 actualTime  # 控制器实际上会完成指令的时间, 会尽量按照 desireTime 来执行, 单位: s

string message      # 该服务中会反馈详细信息, 可用于分析下发失败的原因
```

**规划器配置索引设置**  

- **索引 0**: 底盘世界系位置运动, 自由度为 3, 分别表示 x, y, yaw, 各维度单位分别为: m, m, rad;  
- **索引 1**: 底盘局部系位置运动, 自由度为 3, 分别表示 x, y, yaw, 各维度单位分别为: m, m, rad;  
- **索引 2**: 躯干笛卡尔局部系运动, 自由度为 4, 分别表示 x, z, yaw, pitch, 各维度单位分别为: m, m, rad, rad;  
- **索引 3**: 下肢关节运动, 自由度为 4, 分别表示下肢4个关节的角度, 各维度单位为: rad;
- **索引 4**: 左臂笛卡尔世界系运动, 自由度为 6, 分别表示左臂的6d位姿, 顺序为: x,y,z,yaw,pitch,roll, 各维度单位为: m, m, m, rad, rad, rad;
- **索引 5**: 右臂笛卡尔世界系运动, 自由度为 6, 分别表示右臂的6d位姿, 顺序为: x,y,z,yaw,pitch,roll, 各维度单位为: m, m, m, rad, rad, rad;
- **索引 6**: 左臂笛卡尔局部系运动, 自由度为 6, 分别表示左臂的6d位姿, 顺序为: x,y,z,yaw,pitch,roll, 各维度单位为: m, m, m, rad, rad, rad;
- **索引 7**: 右臂笛卡尔局部系运动, 自由度为 6, 分别表示右臂的6d位姿, 顺序为: x,y,z,yaw,pitch,roll, 各维度单位为: m, m, m, rad, rad, rad;
- **索引 8**: 左臂上肢关节运动, 自由度为 7, 分别表示左臂7个关节的角度, 各维度单位为: rad;
- **索引 9**: 右臂上肢关节运动, 自由度为 7, 分别表示右臂7个关节的角度, 各维度单位为: rad;

#### 5. `/mobile_manipulator_timed_multi_cmd` - 设置多个规划器的基于时间的期望指令向量

**请求字段：**
```yaml
kuavo_msgs/timedSingleCmd[] timedCmdVec  # 时间指令的结构体数组, 支持排列多个规划器指令

bool isSync   # 多个规划器是否做时间同步: false 则表示不做同步, 按各自预设执行; true 则各规划器采用最长的执行时间作为期望 
int8 planner_index   # 规划器索引, 具体看下面说明, 与 /mobile_manipulator_set_ruckig_planner_params 不相同

float32 desireTime   # 期望执行时间, 单位: s, 如果 desireTime 在物理约束内, 则严格按 desireTime 设置轨迹
```

#### 7. `/mobile_manipulator_timed_offline_traj` - 设置躯干与双臂笛卡尔的离线轨迹缓存

**服务类型：** `kuavo_msgs/lbMultiTimedOfflineTraj`

**功能描述：** 用于下发躯干和双臂的笛卡尔离线轨迹，仅更新缓存，并不直接执行

**请求字段：**
```yaml
kuavo_msgs/lbTimedOfflineTraj[] offlineTraj   # 离线轨迹的结构体数组, 支持排列多个规划器指令
  kuavo_msgs/timedPoint[] timedTraj   # 离线轨迹的缓存数组
    float32 desireTime                # 期望执行时间, 从使能执行后开始计时, 单位: s, 必须包含时间0, 且保证时间单调递增, 否则会报错
    float64[] cmdVec                  # 指令向量, 必须与所调用的规划器的自由度一致, 否则会报错
  int8 frame                          # 仅双臂笛卡尔生效, 用于指定笛卡尔轨迹所在的坐标系, 0: 世界系, 1: 局部系
  int8 plannerIndex                   # 轨迹索引, 0: 左臂笛卡尔, 1: 右臂笛卡尔, 2: 躯干笛卡尔
```

**响应字段：**
```yaml
bool isSuccess      # 指令下发是否成功, True 表示下发成功
string message      # 该服务中会反馈详细信息, 可用于分析下发失败的原因
```

**相关演示脚本：** 
- `src/demo/test_kuavo_wheel_real/cmd_offline_traj_test.py`
- `lb_ctrl_api.py` 中的 `set_lb_multi_timed_offline_traj()`

#### 7. `/mobile_manipulator_timed_offline_traj_enable` - 启动或关闭离线轨迹执行

**服务类型：** `std_srvs/SetBool`

**功能描述：** 用于启动离线轨迹执行

**请求字段：**
```yaml
bool data    # 设置为 True, 可使能机器人执行离线轨迹缓存中的轨迹, 此时常规话题或服务下发指令会失效;
             # 设置为 False, 失能离线轨迹执行模式，常规话题或服务可正常执行;
```

**响应字段：**
```yaml
bool success        # 使能或失能是否成功
string message      # 详细信息, 用于确认设置为使能还是失能
```

**相关演示脚本：** 
- `src/demo/test_kuavo_wheel_real/cmd_offline_traj_test.py`
- `lb_ctrl_api.py` 中的 `set_offline_trajectory_enable()`
