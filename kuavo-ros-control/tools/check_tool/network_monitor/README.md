# 上下位机 网络连接监控

#### 系统架构
- **上位机**：运行服务端（`shangweiji_server.py`），负责接收并响应ping请求
- **下位机**：运行客户端（`xiaweiji_client.py`），负责发送ping请求并收集统计数据

## 重要提醒

**⚠️ 强烈建议通过VNC桌面进行操作**

- 避免在本地电脑终端运行程序，当本地电脑与机器人断连时，终端会退出，导致监听中断
- 通过VNC连接操作，只有在上下位机设备关机时程序才会退出

## 安装与配置
### 1. 启动程序

**⚠️ 重要：请通过VNC桌面进行操作，确保程序稳定运行**

#### 启动顺序

> **请先启动服务端，再启动客户端**

**步骤1：在上位机启动服务端**

```bash
./network_monitor.sh
```
选择 **启动上位机监测**

**步骤2：在下位机启动客户端**

```bash
network_monitor.sh
```
选择 **启动下位机监测**

### 2. 程序退出

**正确的退出顺序：**
1. 先在下位机客户端按 `Ctrl+C` 退出
2. 再在上位机服务端按 `Ctrl+C` 退出

> **提示**：上位机服务端可以保持运行状态，从开机后一直运行。日常操作中只需要启停下位机的客户端即可。

### 3. 参数配置

在 `xiaweiji_client.py` 文件顶部可以配置以下参数：

```python
# 网络监控配置
FREQUENCY = 10                    # 发送频率 (Hz)
TIMEOUT = 1.0                     # 请求超时时间 (秒)
INTERVAL = 3600                   # 阶段报告间隔 (秒)

# 文件生成配置
GENERATE_HOURLY_FILES = True      # 是否生成单独的阶段报告文件
GENERATE_FINAL_FILE = True        # 是否生成单独的最终报告文件
```

#### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `FREQUENCY` | 控制ping请求的发送频率 | 10Hz（每秒10次） |
| `TIMEOUT` | 请求超时时间，超过此时间未收到响应则判定为超时 | 1.0秒 |
| `INTERVAL` | 阶段报告生成间隔 | 3600秒（1小时） |
| `GENERATE_HOURLY_FILES` | 是否生成单独的阶段报告文件 | True |
| `GENERATE_FINAL_FILE` | 是否生成单独的最终报告文件 | True |

## 输出文件

程序运行时会自动生成以下文件：

- **日志文件**：`output/logs/network_monitor_YYYYMMDD_HHMMSS.log`
- **统计报告**：`output/logs/network_monitor_reports_YYYYMMDD_HHMMSS.txt`
- **CSV数据**：`output/csv/network_data_YYYYMMDD_HHMMSS.csv`
- **阶段报告**：`output/logs/final_report_YYYYMMDD_HHMMSS_to_YYYYMMDD_HHMMSS.txt`
