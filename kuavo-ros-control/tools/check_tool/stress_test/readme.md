# CPU 压力 & 温度 & 频率稳定性测试

## 脚本说明

本目录包含两个压力测试脚本，根据不同的测试需求选择使用：

| 脚本 | 说明 |
|------|------|
| `stress_test_normal_cores.sh` | 使用 stress 对所有未隔离的 CPU 核心进行压力测试
| `stress_test_all_cores.sh` | 使用 stress 和 stress-ng 对所有核心进行压力测试

## 启动步骤

进入当前目录，打开终端。

### 满载测试（推荐）

```bash
./stress_test_all_cores.sh
```

### 普通测试

```bash
./stress_test_normal_cores.sh
```

## 使用说明

### 1. 机型自动检测

脚本会根据 `ROBOT_VERSION` 环境变量自动判断机型：

| 版本号范围 | 机型 | 温度阈值 |
|-----------|------|---------|
| 50~59 | kuavo5 | 80°C |
| 60~69 | kuavo5w | 80°C |
| 13~17 | roban | 90°C |

如果无法自动检测，脚本会提示手动选择机型。

### 2. 设置测试时长

直接回车：使用默认时长（10分钟）

输入数字后回车：使用自定义时长（秒）

### 3. 测试过程中操作

- 按 `q` 键：安全退出测试
- Ctrl+C：强制退出

## 结果判断

测试结束后根据结果显示：

- ✅ **测试通过**（绿色）：温度和频率都在正常范围内
- ❌ **测试未通过**（红色）：温度超过阈值或存在低频核心

## 日志文件

日志文件保存文件名格式：`stress_test_log_YYYYMMDD_HHMMSS.log`

## 测试参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 温度阈值 | 80°C/90°C | 根据机型自动设置 |
| 低频阈值 | 3000 MHz | 低于此频率判定为低频 |
| 低频超时 | 50秒 | 低频持续时间超过此值判定失败 |
| 温度超时 | 50秒 | 温度超过阈值持续时间超过此值判定失败 |

## 注意事项

1. 测试过程中 CPU 会满载运行，温度会升高，请确保散热良好
2. 低频统计在启动5秒后才开始，避免启动初期不稳定
3. 建议在使用电源适配器的情况下进行测试
