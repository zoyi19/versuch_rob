# yolo识别及抓取简易说明文档

## 项目说明:
- `yolo_object_capture`文件夹位于下位机的kuavo-ros-opensource/src/demo/目录下
- `detection_yolo_v8`文件夹位于上位机的kuavo_ros_application/src/ros_vision目录下

- 在上位机编译:
```bash
cd kuavo_ros_application
catkin build detection_yolo_v8
```
- 在下位机安装:
```bash
sudo apt-get install ros-noetic-vision-msgs
```

## 参数修改
1. 需要提前输入场景参数
- 配置文件~/kuavo_ros_application/src/ros_vision/detection_yolo_v8/config/params.yaml
```bash
# 高度参数(单位: 米)
height_table: 0.83
height_bottle: 0.22
height_cup: 0.16
height_orange: 0.071
```
- 如果是抓取水瓶, 将height_table改为实际桌面高度, 将height_bottle改为实际水瓶高度即可
- 如果是抓取橙子, 将height_table改为实际桌面高度, 将height_orange改为实际橙子高度即可

2. 另,如果发现检测结果的置信度较低, 可适当降低置信度阈值
- 配置文件同样为detection_yolo_v8/config/params.yaml
- 默认conf_threshold值为0.5, 可适当降低该值

## 运行流程
**启动前简易参考文档网站进行头部和手臂限位自动标定:https://kuavo.lejurobot.com/manual/basic_usage/kuavo-ros-control/docs/3%E8%B0%83%E8%AF%95%E6%95%99%E7%A8%8B/%E6%9C%BA%E5%99%A8%E4%BA%BA%E5%85%B3%E8%8A%82%E6%A0%87%E5%AE%9A/#%E5%A4%B4%E9%83%A8%E5%92%8C%E6%89%8B%E8%87%82%E9%9B%B6%E7%82%B9%E8%87%AA%E5%8A%A8%E6%A0%87%E5%AE%9A**
1. 下位机启动机器人, 让机器人站立(已用遥控器使机器人站立可跳过此步)
2. 上位机启动相机
```bash
cd kuavo_ros_application
source devel/setup.bash
roslaunch dynamic_biped kuavo5_sensor_only_enable.launch
```
3. 上位机启动yolo识别线程
```bash
cd kuavo_ros_application
source devel/setup.bash
rosrun detection_yolo_v8 detection.launch
```
4. 可通过rqt_image_view查看图像识别结果
5. 可通过rostopic echo /robot_yolov8_info查看坐标转换结果
6. 下位机启动抓取水瓶demo
**确保步骤1-3已正常运行**
```bash
cd kuavo-ros-opensource
source devel/setup.bash
# 以下为运行示例, 这里的两个参数不存在绑定关系, 只加其中一个参数也可以正常运行
# 带抓取偏移量, 自动转腰(推荐)
python3 src/demo/yolo_object_capture/yolo_cylinder_capture.py --use_offset
# 不带抓取偏移量, 不带转腰
python3 src/demo/yolo_object_capture/yolo_cylinder_capture.py --no_waist
```
7. 下位机启动抓取橙子demo
**确保步骤1-3已正常运行**
```bash
cd kuavo-ros-opensource
source devel/setup.bash
# 以下为运行示例, 这里的两个参数不存在绑定关系, 只加其中一个参数也可以正常运行
# 带抓取偏移量, 自动转腰(推荐)
python3 src/demo/yolo_object_capture/yolo_sphere_capture.py --use_offset
# 不带抓取偏移量, 不带转腰
python3 src/demo/yolo_object_capture/yolo_sphere_capture.py --no_waist
```

## 补充:
- 转腰逻辑
  - 若不加参数"--no_waist",程序会先进行转腰的尝试,若计算出合理的转腰角度则进行"转腰抓取",否则进行"不转腰抓取"

- 抓取策略区别
  - yolo_cylinder_capture.py 是针对水瓶类圆柱形物体, 采用从后向前的抓取策略
  - yolo_sphere_capture.py 是针对小圆形物体, 采用从上到下的抓取策略

- yolo_object_capture/config/offset.yaml(下位机)
  - 这个是抓取demo的偏移量配置文件,加入参数"--use_offset"后会启用
  - 根据注释修改你期望抓取的物体即可,参考坐标系为base_link坐标系(右手系, 机器人面朝方向为x正方向, 机器人上方为z方向)
  - offset:该参数用于对ik逆解的输入xyz进行修正, 以及对期望的yaw姿态进行缩放
  - offset_angle:该参数用于对ik逆解的期望姿态中的yaw角进行调整
  - 如抓取瓶子时, 可适当把"offset_angle"调小,如0.1, 这样可以尽量保证末端姿态朝向正前方
