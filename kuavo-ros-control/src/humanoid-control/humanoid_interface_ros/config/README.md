# 转向区间配置文件说明

## 文件位置
`config/step_turn.yaml`

## 功能说明
此配置文件用于设置机器人单步转向控制的区间参数。系统会根据摇杆的输入值，自动选择对应的转向区间和轨迹。

## 配置原理

### 对称性设计
- **只需配置左转区间**：参数文件中只需要设置左转的参数
- **自动生成右转**：代码会自动根据左转参数生成对称的右转区间

### 对称规则
左转区间配置后，右转区间按以下规则自动生成：
- **摇杆区间范围**：取反（负值变正值）
  - 左转: `-0.15 ~ 0.0` → 右转: `0.0 ~ 0.15`
- **body_yaw**：取反（正变负，负变正）

## 参数说明

### turn_zones_count
转向区间的数量（只计算左转区间数量，右转会自动生成）

### left_turn_zones
左转区间配置列表，每个区间包含以下参数：

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `min_value` | float | 摇杆区间最小值（负值，范围 -1.0 ~ 0.0） |
| `max_value` | float | 摇杆区间最大值（负值，范围 -1.0 ~ 0.0） |
| `body_yaw` | double | 身体偏航角（度，正值表示左转） |

> **注意**：`body_x`、`body_y`、`body_z` 参数已在代码中固定为 0，无需在配置文件中设置。

## 配置示例

```yaml
turn_zones_count: 4

left_turn_zones:
  # 小角度左转: 摇杆范围 -0.15 ~ 0.0，转向角度 10°
  - min_value: -0.15
    max_value: 0.0
    body_yaw: 10.0

  # 小角度左转: 摇杆范围 -0.45 ~ -0.15，转向角度 20°
  - min_value: -0.45
    max_value: -0.15
    body_yaw: 20.0

  # 中角度左转: 摇杆范围 -0.75 ~ -0.451，转向角度 30°
  - min_value: -0.75
    max_value: -0.451
    body_yaw: 30.0

  # 大角度左转: 摇杆范围 -1.0 ~ -0.751，转向角度 45°
  - min_value: -1.0
    max_value: -0.751
    body_yaw: 45.0
```

## 生成结果示例

上述配置会生成以下转向区间：

**右转区间**（自动生成）：
- 区间0: `0.0 ~ 0.15`，pose=`[0, 0, 0, -10.0]`
- 区间1: `0.15 ~ 0.45`，pose=`[0, 0, 0, -20.0]`
- 区间2: `0.45 ~ 0.75`，pose=`[0, 0, 0, -30.0]`
- 区间3: `0.75 ~ 1.0`，pose=`[0, 0, 0, -45.0]`

**左转区间**（从配置文件读取）：
- 区间0: `-0.15 ~ 0.0`，pose=`[0, 0, 0, 10.0]`
- 区间1: `-0.45 ~ -0.15`，pose=`[0, 0, 0, 20.0]`
- 区间2: `-0.75 ~ -0.451`，pose=`[0, 0, 0, 30.0]`
- 区间3: `-1.0 ~ -0.751`，pose=`[0, 0, 0, 45.0]`

## 修改配置

1. 编辑 `config/step_turn.yaml` 文件
2. 只需修改左转区间的参数（`min_value`, `max_value`, `body_yaw`）
3. 重启节点，配置会自动加载
4. 如果 YAML 文件加载失败，系统会使用默认配置

## 注意事项

1. **区间连续性**：建议配置的区间相互衔接，避免出现空白区间
2. **摇杆范围**：左转区间应在 `-1.0 ~ 0.0` 范围内
3. **参数单位**：角度单位为度
4. **备用方案**：如果 YAML 文件读取失败，系统会自动使用内置的默认配置
