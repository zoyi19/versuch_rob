# Kuavo IK 项目现况说明

> 更新日期：2026-02-13

---

## 一、项目概述

本项目基于 **kuavo_ik** 逆运动学库，实现了从 **UMI 遥操作演示数据** 到 **Kuavo 机器人右臂关节角** 的完整处理流水线。

**核心目标**：将人类使用 UMI 手柄的遥操作轨迹，经过坐标变换后，验证其在 Kuavo 机器人右臂上的 IK（逆运动学）可达性。

### 整体数据流

```
ROS Bag (UMI演示录制)
  ├─ /camera/color/image_raw   (D405 手部相机)
  ├─ /orbbec/rgb/image_raw     (Orbbec 头部相机)
  ├─ /umi_odom                 (UMI 手柄里程计)
  └─ /tf                       (mocap_frame → mocap_umi, mocap_orbbec)
         │
         ▼
  ① 数据对齐 (bag_data_alignment.py)
         │
         ▼
  ② 坐标变换 (coordinate_transform.py)
     mocap坐标系 → waist_yaw_link坐标系
         │
         ▼
  ③ IK可达性验证 (verify_ik_trajectory.py)
     逐帧 IKAnalytical 求解 + FKTool 验证
         │
         ▼
  输出: 关节角轨迹 + 误差统计 + 可视化
```

## 三、核心思路与算法

### 3.1 坐标变换链

```
T(waist_yaw → mocap_umi) = T(waist_yaw → zhead_2) × T(mocap_orbbec → mocap_umi)
```

- **T(waist_yaw → zhead_2)**：从机器人 bag 的 `/tf` 提取，头部固定为常数
  - 变换路径：`waist_yaw_link → zhead_1_link → zhead_2_link`
  - 实测值：位置 (-0.0171, 0.0000, 0.4050) m，pitch ≈ 20.47°

- **T(mocap_orbbec → mocap_umi)**：从演示 bag 的 `/tf` 逐帧计算
  - = T(mocap_frame → mocap_orbbec)⁻¹ × T(mocap_frame → mocap_umi)
  - mocap_orbbec 近似静态（std < 1mm），使用平均值

- **关键假设（方案A）**：`mocap_orbbec` 坐标系 ≡ `zhead_2_link` 坐标系（标定时有意对齐），即 T(zhead_2 → mocap_orbbec) = I（单位矩阵）

- **设计原因**：使用相对头部（Orbbec）的坐标而非世界坐标，是为了规避两次录制时世界坐标系原点不同的问题。

### 3.2 IK 求解

- **求解器**：`IKAnalytical`（解析解），速度快（>400 fps）
- **机器人型号**：`model_type='60'`（base_link = waist_yaw_link）
- **末端执行器**：`zarm_r7_link`（右臂）
- **对应关系**：`mocap_umi` 直接对应 `zarm_r7_link`

### 3.3 FK 验证中的坐标系补偿

由于 `FKTool` 使用的 URDF 是 `model_type='46'`，而 IK 使用 `model_type='60'`，两者的 `l1_link_in_base` 偏移不同：

| 参数 | model_type='46' (URDF/FK) | model_type='60' (IK) |
|------|---------------------------|----------------------|
| 左臂 l1_link_in_base | [-0.0175, 0.2927, 0.445] | [-0.018, 0.255, 0.192] |
| 右臂 l1_link_in_base | [-0.0175, -0.2927, 0.445] | [-0.018, -0.255, 0.192] |

FK 验证时需要补偿此偏移：

```python
FK_OFFSET_COMPENSATION = L1_URDF_RIGHT - L1_M60_RIGHT
fk_pos_corrected = fk_pos - FK_OFFSET_COMPENSATION
```

### 3.4 姿态误差计算

使用四元数测地距离（而非欧拉角差值），避免 wrap-around 问题：

```python
R_diff = R_fk * R_target.inv()
rot_error_rad = R_diff.magnitude()
```

## 四、当前验证结果

### 4.1 数据概况

- **数据源**：4 个 ROS Bag（umi_data/ 目录下）
- **录制频率**：120 Hz
- **每个 Bag 处理独立**，结果互不混合

### 4.2 IK 验证结果汇总

| Bag | 帧数 | IK成功率 | 位置误差(mm) | 姿态误差(°) |
|-----|------|----------|-------------|-------------|
| bag_0 (18-45-56) | 1699 | **100%** | mean=16.41 | mean=101.63 |
| bag_1 (18-46-17) | 1233 | **100%** | mean=14.71 | mean=107.44 |
| bag_2 (18-46-30) | 1529 | **100%** | mean=15.66 | mean=95.22 |
| bag_3 (18-46-46) | 1803 | **100%** | mean=16.21 | mean=100.69 |

### 4.3 结果分析

- ✅ **IK 成功率 100%**：所有帧均可求解，说明演示轨迹的位置完全在机器人工作空间内
- ⚠️ **位置误差 ~15mm**：由 model_type='46' URDF 与 model_type='60' IK 参数的固有差异导致，已做偏移补偿但仍有残差
- ⚠️ **姿态误差 ~100°**：主要原因是 `mocap_umi` 标记物的朝向与 `zarm_r7_link` 的默认坐标系朝向存在较大差异（需要标定旋转矩阵）
- ⚠️ **关节限位警告 100%**：joint_5 和 joint_7 频繁触及限位边界

### 5.3 在自己的代码中使用 IK

```python
import numpy as np
from scipy.spatial.transform import Rotation as R
from kuavo_ik.ik_library import IKAnalytical
from kuavo_ik.fk_tool import FKTool

# IK求解
pos = np.array([0.4, 0.3, 0.5])
quat = R.from_euler('xyz', [0, -np.pi/2, 0]).as_quat()  # [qx, qy, qz, qw]

joint_angles = IKAnalytical.compute(
    eef_pos=pos,
    eef_quat_xyzw=quat,
    eef_frame='zarm_r7_link',   # 右臂
    model_type='60',             # waist_yaw_link 为基座
    limit=True                   # 启用关节限位
)

# FK验证
fk_tool = FKTool()
full_joints = np.concatenate([np.zeros(7), joint_angles])  # [左臂0, 右臂joints]
fk_pos, fk_quat = fk_tool.compute(full_joints, frame_name='zarm_r7_link')
```

## 八、已知问题与后续方向

### 已知问题

1. **姿态误差大（~100°）**：`mocap_umi` 标记物与 `zarm_r7_link` 坐标系朝向不一致，需要标定一个固定旋转矩阵 `T(zarm_r7 → mocap_umi_local)` 来补偿
2. **FK 位置误差 ~15mm**：model_type='46' URDF 与 model_type='60' IK 参数的固有差异，已做线性补偿但有残差
3. **关节频繁触限**：joint_5（±90°）和 joint_7（-110°）几乎持续在限位边界，实际执行需关注

### 可能的后续方向

- 标定 `mocap_umi` 到 `zarm_r7_link` 的姿态偏移旋转矩阵
- 使用匹配的 model_type='60' URDF 模型做 FK 验证（如果有）
- 轨迹平滑处理，减少关节角跳变
- 添加碰撞检测和路径规划
- 集成到实际机器人控制流程
